Docker != контейнер. В данном сабже Docker - система управления контейнерами. Внутри самого Docker находится много всего, когда запускаем контейнер через докер, то по факту, сначала запускается демон докера, потом управление передается утилите containered (контролирует работу контейнеров, работу с репозиторием контейнеров и тд), которая управляет работой контейнеров через runc - низкоуровневую среду выполнения контейнеров.

Kubernetes - система оркестрации контейнеров. Для работы с контейнерами он поддерживает стандарт  **Container Runtime Interface (CRI)** - определяет API между Kubernetes и Container Runtime (средой выполнения контейнеров).

То, что многие люди называют Docker—образами, на самом деле является образами в формате Open Container Initiative (OCI).
